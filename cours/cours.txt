STRUCTURE DES ORDINATEURS

ordinateur -> UAL (Unité Arithmetique et Logique (ALU en anglais) -> Unité de contrôle puis Entrées / Sorties
opérations -> opérateurs {0, 1} -> +, -, x -> propriétés / fonctions / tables de vérité -> simplification de circuits
pour la transformation d’états inderterminés, s’en servir pour simplifier les fonctions avec le 1 le plus souvent
lorsque l’on a plus de bits que nécéssaires, ceux en trop sont dans un état indéterminés (optimisation de bits, de place)
voici pourquoi on simplifie les fonctions, pour les récupérer, les “économiser” (voir photo tel)
FND -> SOMME = 1
FNC -> SOMME = 0

binaire = code pondéré (rappel de la formule binomiale avec Somme de i de Bi x 2^i)
/-> code BCD (Binary Coded Decimal) -> code seulement les 10 chiffres // 
/-> code de Aiken -> pratique, pondéré, BCDique, réfléchi, autocomplémentaire

codes non-pondérés:
/-> code excédent 3 (+3) -> code BCDique -> on prend le binaire et on rajoute 3, (0 + 3, 1 + 3 etc…)
/-> code de GRAY -> on commence à 0 : première valeur c’est 0, valeur suivante on inverse le bit le plus à droite (sans retomber sur une valeur déjà connue)

codes redondants - détection d’erreurs -> bit de parité (afin d’effectuer des tests de communication entre machines)
pour déterminer la parité d’une valeur en binaire, on fait un OU EXCLUSIF (XOR) entre chaque valeur (cumulatif / au fur et à mesure)
(ex barette mémoire avec détection d’erreurs avec ce fonctionnement)
pour le code pondéré redondant -> utilisation de 5 bits (2^5 -> 32) 
/-> chaque code de valeurs ne contient que des 1 donc si il n’y a pas 2 x 1, on peut détecter une erreur

/-> code ASCII (American Standart Convention for Information Interchange) -> codé sur 2^7 bits (128 valeurs) 
+ 1 bit de parité (8 bits) ou ce bit utilisé pour un code ascii étendu (variable selon les pays pour les caractères différents)

DANS UN CIRCUIT: 
on a le transcodeur : circuit dans lequel on a le codeur (pas de code -> code) et le décodeur (code -> pas de code)
le transcodeur fait donc code -> code

COTÉ CODEUR: 1 et 1 seule entrée à 1
/-> “pas de code”: 4 valeurs : on a 4 touches : 0, 1, 2, 3 : si on appuie sur la première touche, le circuit de codage doit générer 0, etc…
/-> on fait un circuit avec 4 entrées, 2 sorties et un tableau avec 2^4 -> 16 lignes, 2 fonctions, et toute les autres valeurs que 0, 1, 2 et 3, il -/-> n’y a ni 0 ni 1, cela ne nous interesse pas
/-> En sortie, le codeur sort la valeur qui nous intérèsse, celle que l’on a mis à 1 (pression) (cf photo tableau)

COTÉ DÉCODEUR:
/-> prend en entrée 1 valeur codée dans un code
/-> prend en sortie, la sortie activée dont le numéro correspond à la valeur d'entrée
/-> exemple de décodeur binaire sur 3 bits, 8 sorties (cf photo)

COTÉ TRANSCODEUR
/-> prend en entrée 1 valeur dans 1 code
/-> prend en sortie la même valeur dans un autre code
/-> exemple code GRAY sur 3 bits -> binaire

CIRCULATION D’INFORMATION
/-> bus : fils dans lesquels circule l’information : 1 fil = 1 bit
/->plusieurs bits pour une information tous en même temps : 
bus parallèle (exemple DATA) ou l’un après l’autre : bus série : par exemple l’USB, SATA
/-> parallèle plus rapide mais plus cher car plus de fils
/-> série = moins vite = moins cher
